# خبرگیر روزانه (News Fetcher)

این پروژه هر روز به‌صورت خودکار از سایت‌هایی که معرفی می‌کنی، خبرهای جدید (عنوان + لینک + خلاصه) را جمع‌آوری کرده و از طریق یک ربات تلگرام برایت ارسال می‌کند.

## ساختار فایل‌ها

```
news-fetcher/
├── sites.txt                     # لیست آدرس سایت‌ها (یک سایت در هر خط)
├── news_fetcher.py                # اسکریپت اصلی
├── requirements.txt               # وابستگی‌های پایتون
├── state/sent_links.json          # حافظه‌ی لینک‌های قبلاً ارسال‌شده (خودکار به‌روز می‌شود)
└── .github/workflows/daily-news.yml  # تعریف اجرای خودکار روزانه در گیت‌هاب
```

## مرحله ۱: ساخت ربات تلگرام و گرفتن توکن

1. در تلگرام به [@BotFather](https://t.me/BotFather) پیام بده.
2. دستور `/newbot` را بفرست و اسم و یوزرنیم دلخواه برای ربات انتخاب کن.
3. BotFather یک **توکن** به شکل `123456789:ABCdefGhIJKlmnoPQRstuVWxyz` به تو می‌دهد. این را جایی نگه دار (`TELEGRAM_BOT_TOKEN`).

## مرحله ۲: گرفتن Chat ID

1. یک پیام (مثلاً `سلام`) برای ربات تازه‌ساخته‌ات بفرست (باید اول با آن استارت کنی: `/start`).
2. در مرورگر این آدرس را باز کن (به‌جای `<TOKEN>` توکن خودت را بگذار):
   `https://api.telegram.org/bot<TOKEN>/getUpdates`
3. در پاسخ JSON، دنبال مقدار `"chat":{"id": ...}` بگرد. همان عدد، `TELEGRAM_CHAT_ID` توست.

## مرحله ۳: ساخت ریپازیتوری در گیت‌هاب

1. یک ریپازیتوری جدید (می‌تواند خصوصی/Private باشد) در گیت‌هاب بساز.
2. تمام فایل‌های این پوشه را در آن ریپازیتوری Push کن.

## مرحله ۴: تنظیم Secrets در گیت‌هاب

در ریپازیتوری برو به:
`Settings > Secrets and variables > Actions > New repository secret`

دو مقدار زیر را اضافه کن:
- `TELEGRAM_BOT_TOKEN` → توکن ربات
- `TELEGRAM_CHAT_ID` → آی‌دی چتی که مرحله قبل گرفتی

## مرحله ۵: اضافه کردن سایت‌ها

فایل `sites.txt` را باز کن و آدرس سایت‌های موردنظرت را (هر کدام در یک خط) اضافه کن:

```
https://www.isna.ir
https://www.mehrnews.com
```

## مرحله ۶: اجرا

- به‌صورت خودکار هر روز ساعت ۹ صبح (به وقت ایران) اجرا می‌شود.
- برای اجرای دستی و تست فوری: در ریپازیتوری برو به تب **Actions** → روی workflow با نام **Daily News Fetcher** کلیک کن → دکمه **Run workflow**.

## نکات مهم

- بعضی سایت‌ها ساختار HTML خاصی دارند که ممکن است الگوریتم اسکرپ عمومی نتواند خبرهایشان را کامل تشخیص دهد. اگر برای سایت خاصی نتیجه ضعیف بود، بگو تا الگوریتم استخراج را مخصوص همان سایت تنظیم کنم.
- بعضی سایت‌ها اسکرپ خودکار را محدود می‌کنند (مثلاً با Cloudflare)؛ برای چنین سایت‌هایی ممکن است نیاز به روش دیگری باشد.
- فایل `state/sent_links.json` حافظه‌ی برنامه است؛ اگر پاکش کنی، همه‌ی خبرهای موجود دوباره به‌عنوان «جدید» ارسال می‌شوند.
- تست محلی (روی کامپیوتر خودت) با دستورهای زیر:

```bash
pip install -r requirements.txt
export TELEGRAM_BOT_TOKEN="توکن_ربات"
export TELEGRAM_CHAT_ID="آیدی_چت"
python news_fetcher.py
```
